The Attack Flow — A Demo in Six Steps
1
Choose a Scenario
Select from 24 production attacks across Endpoint, NDR, and Firewall. Each scenario includes expected detections, MITRE techniques, and a pre-built runbook.
2
Target the Victim VM
Enter the Windows VM IP address and click Launch. The sandcat agent is pre-deployed at first boot on every LabOps VM — no manual setup needed.
3
CALDERA Orchestrates the Attack
The CALDERA C2 framework dispatches abilities to the agent. The SE narrates the story — they don't operate the attack.
4
Real Techniques Execute Live
Actual PowerShell, LOLBins, credential tools, and lateral movement run on the victim VM — visible in the CALDERA feed as they happen.
5
Sophos Detects and Alerts
Endpoint, NDR, or Firewall generates real telemetry and alerts. The customer watches their product respond to a live attack in real time.
6
SE Pivots to Sophos Central
Open the XDR threat hunt and walk the customer through the kill-chain investigation — turning a live attack into a proof of value.

LabOps Highlights
VM Provisioning
Clone any Windows template from Proxmox in 2–4 minutes. Terraform handles the clone; Ansible configures RDP, creates the demo user, and disables the Windows Firewall.
2–4 minutes · fully automated
Start · Stop · Destroy
Full VM lifecycle from the browser. Stop VMs between sessions to conserve resources. Destroy to reclaim disk after a demo. Always a confirm dialog before destruction.
graceful shutdown or force-off
Browser-Based RDP
Apache Guacamole renders a full Windows desktop in any browser tab. No VPN, no RDP client, no firewall exceptions. Share the URL — anyone can connect instantly.
no client software required
Service Health Monitor

Terraform + Ansible
